# Maratonas de Programação 2026

## Por que participar?
* Empresas valorizam participação em maratona.
* Etapas de processo seletivo incluem algoritmos e estrutura de dados (Nubank, Google, Meta, Uber, iFood).
* Bom para desenvolver habilidades em programação, trabalho em equipe, etc.

## Como funciona uma Maratona?
* **Duração:** 5 horas de prova corrida.
* **Equipe:** 3 alunos por time.
* **Desafio:** Caderno com 8 a 14 problemas.
* **Objetivo:** Resolver a maior quantidade de problemas.
* **Julgamento:** O julgamento é automático e se divide em 3 vereditos principais:
    * *Accepted* (Passou)
    * *Wrong Answer* (Resposta Errada)
    * *Time Limit Exceeded* (Lento demais)

## Preparação de Conteúdos
**Conteúdo em slides:**
* C++
* Algoritmo e Estrutura de dados
* Grafos
* Programação dinâmica

**Foco em resolver problemas:**
* Analisar um problema.
* Observar as soluções dos alunos.
* Apresentar a melhor solução e como chegar lá.

**Metodologia:**
* 1 encontro semanal para contest próprio da equipe.
* Treinar o trabalho em equipe e entendimento da solução do editorial.
* Focar na autonomia dos próprios alunos em ver os conteúdos, tirar dúvidas, etc.
* **Meta:** 30 problemas do Codeforces/Beecrowd por mês para cada aluno.
* Ranking dos alunos.
